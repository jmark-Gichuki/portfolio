üß© Lab 2:Using Wireshark to View Network Traffic

Problem:

Capture and analyze both local and remote ICMP traffic using Wireshark to understand how packets are transmitted, addressed, and resolved across network layers.

Tools Used:

Wireshark, Command Prompt, TCP/IP utilities (ping, ipconfig)

Approach:

Started by retrieving interface addresses using ipconfig /all to identify local network configurations. Initiated Wireshark and selected the active network interface to begin packet capture. Applied filters to isolate ICMP traffic and performed pings to both the local gateway (192.168.100.1) and remote hosts such as www.yahoo.com, www.cisco.com, and www.google.com. Analyzed captured packets to compare local versus remote traffic. Observed that local communication displayed both source and destination MAC addresses (e.g., PC-to-router), while remote traffic only displayed the default gateway‚Äôs MAC address. Examined how DNS resolved hostnames into IP addresses and how ARP resolved local device MACs before ICMP packets were sent.

Key Lessons:

This lab reinforced the difference between local (Layer 2) and remote (Layer 3) communication. When communicating within a LAN, devices use physical MAC addressing, while remote communication relies on the default gateway to forward packets. Learned how Wireshark can visualize packet flow, ARP resolution, and DNS lookups, enhancing network troubleshooting and analysis skills.

üß© Lab 3:Build a Switch and Router Network

Problem:

Design, configure, and verify a basic network consisting of a router, switch, and two end devices using Cisco Packet Tracer. The objective was to establish connectivity between subnets by assigning IP addresses, enabling interfaces, and testing communication using commands like ping, show ip route, and show ip interface brief.

Tools Used:

Cisco Packet Tracer (Simulation Mode), Cisco IOS Command-Line Interface

Approach:

Set up the network topology with a router, switch, and two PCs. Initialized and configured devices by assigning static IPv4 and IPv6 addresses to interfaces. Configured router interfaces (G0/0/0 and G0/0/1) for inter-subnet routing and set up the switch with VLAN interface and default gateway. Verified connectivity by successfully pinging between PCs through the router. Used show commands (show ip route, show ipv6 route, and show interfaces) to confirm interface status, routing entries, and connectivity. Addressed common issues such as ‚Äúadministratively down‚Äù interfaces using the no shutdown command and corrected potential misconfigurations like duplicate IP assignments.

Key Lessons:

This lab provided hands-on experience in network configuration and troubleshooting. It reinforced understanding of static IP addressing, interface configuration, and the relationship between Layer 2 (switching) and Layer 3 (routing). Gained deeper insight into verifying device operations using diagnostic commands and learned how proper IP planning and gateway configuration ensure seamless communication across networks.

Problem:

Explore and understand the fundamentals of network traffic analysis through hands-on exercises using tcpdump and Wireshark. The goal was to learn how to capture, filter, and interpret network packets to detect suspicious activities and understand communication between devices across the OSI and TCP/IP layers.

Tools Used:

Wireshark, tcpdump, Linux Command Line, Hack The Box Academy Platform

Approach:

Accessed the ‚ÄúIntroduction to Network Traffic Analysis (Tier 0)‚Äù module on Hack The Box Academy. The lab covered both theoretical and practical aspects of analyzing network packets. Initially reviewed the OSI and TCP/IP models to understand how data flows through different layers. Practiced using tcpdump to capture and filter network packets, applying switches such as -i, -v, -X, -r, and -w for various capture scenarios. Then imported packet captures into Wireshark for detailed analysis. Applied display and capture filters to isolate protocols such as HTTP, DNS, ICMP, and SMB. Examined three-way handshakes, DNS resolutions, and HTTP requests to understand how communication occurs between clients and servers. Additionally, explored Wireshark‚Äôs advanced features ‚Äî such as stream following, conversation statistics, and protocol breakdowns ‚Äî and practiced identifying suspicious activities and reconstructing files from captured traffic.

Key Lessons:

This lab provided strong foundational skills in network traffic analysis. Gained practical experience using tcpdump and Wireshark to inspect live and recorded traffic, apply protocol filters, and interpret captured packets. Learned the importance of understanding OSI layers, differentiating between connection-oriented and connectionless protocols, and detecting abnormal network behavior.The exercise also emphasized how traffic analysis supports cybersecurity operations, from identifying vulnerabilities and unauthorized access to performing forensic investigations.
