Lab Challenges

Documenting my hands-on learning journey through HackTheBox, CyberShujaa, and other cybersecurity training platforms.

ðŸ§© Lab 1: Network Traffic Analysis

Problem:

Analyze suspicious network packets to identify potential security threats and extract sensitive information from captured traffic.

Tools Used:

Wireshark, tcpdump, Linux command line

Approach:

Captured network traffic using tcpdump and imported the PCAP file into Wireshark for detailed analysis. Applied filters to isolate HTTP traffic and examined packet headers for anomalies. Identified unencrypted credentials being transmitted over the network and traced the source and destination IP addresses. Reconstructed TCP streams to view complete conversations between hosts.

Key Lessons:

Understanding network visibility is crucial for detecting potential intrusions. Learned the importance of protocol analysis and how attackers can exploit unencrypted communications. This lab reinforced the need for encryption (HTTPS, VPNs) in protecting sensitive data transmission.

ðŸ§© Lab 2: HTTP Fundamentals & Web Request Manipulation

Problem:

Understand HTTP request/response cycles and learn to manipulate web requests to interact with web applications programmatically.

Tools Used:

Burp Suite, Browser DevTools, cURL

Approach:

Configured browser to proxy traffic through Burp Suite to intercept and modify HTTP requests. Analyzed GET and POST requests, examined headers (User-Agent, Cookie, Authorization), and understood status codes (200, 404, 500). Practiced modifying request parameters to test different server responses and used cURL to send custom HTTP requests from the command line.

Key Lessons:

Mastering HTTP concepts is foundational for ethical hacking and penetration testing. Understanding how web applications communicate helps identify potential vulnerabilities such as improper input validation, session management flaws, and authentication bypasses.

ðŸ§© Lab 3: Network Scanning & Reconnaissance

Problem:

Perform network reconnaissance to discover active hosts, open ports, and running services on a target network.

Tools Used:

Nmap, Netcat, Ping

Approach:

Started with ping sweeps to identify live hosts on the network. Used Nmap for comprehensive port scanning with various scan types (SYN scan, TCP connect scan, UDP scan). Performed service version detection to identify software running on open ports. Analyzed results to map network topology and potential entry points.

Key Lessons:

Reconnaissance is the first phase of penetration testing. Proper enumeration reveals the attack surface and helps prioritize testing efforts. Learned the importance of stealthy scanning techniques and understanding firewall evasion methods. This lab emphasized the defender's perspective of monitoring for scan activities.

ðŸ§© Lab 4: Linux Privilege Escalation

Problem:

Gain root access on a Linux system by exploiting misconfigurations and privilege escalation vectors.

Tools Used:

Linux command line, LinPEAS, GTFOBins

Approach:

Enumerated system information including kernel version, installed packages, and user privileges. Checked for SUID binaries using find command and identified exploitable binaries. Searched for writable files in sensitive directories and examined cron jobs for potential hijacking. Used sudo -l to check allowed commands and leveraged GTFOBins for privilege escalation techniques.

Key Lessons:

Proper system hardening requires careful permission management and regular security audits. Understanding common privilege escalation vectors helps both attackers and defenders. This lab highlighted the importance of the principle of least privilege and proper configuration management in Linux environments.
